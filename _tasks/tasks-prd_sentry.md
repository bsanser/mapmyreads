## Relevant Files

- `next.config.js` — Must be wrapped with `withSentryConfig()` to enable source map upload at build time
- `sentry.client.config.ts` — New file (auto-generated by wizard). Browser-side SDK init with session replay config
- `sentry.server.config.ts` — New file (auto-generated by wizard). Server/API routes SDK init
- `sentry.edge.config.ts` — New file (auto-generated by wizard). Edge runtime SDK init
- `.env.local` — Add Sentry environment variables (DSN, org, project, auth token)
- `app/page.tsx` — Replace `fetch('/api/logs/enrichment', ...)` with Sentry `captureMessage`
- `app/api/authors/batch-resolve/route.ts` — Add Prisma error capture (`db_unavailable` event)
- `app/api/books/batch-covers/route.ts` — Add Prisma error capture and replace `console.warn` with `captureException`
- `app/api/logs/enrichment/route.ts` — Delete this file once replaced by Sentry events
- `lib/countryDetection.ts` — Add explicit capture for Wikidata 429s (`wikidata_rate_limited`) and other API errors
- `lib/authorCountryService.ts` — Wrap enrichment flow in `Sentry.startSpan()` with cache hit/miss attributes
- `lib/bookCoverService.ts` — Wrap cover fetch flow in `Sentry.startSpan()` and replace `console.warn` with `captureException`
- `lib/csvParser.ts` — Add `captureException` on parse failures
- `lib/storage.ts` — Add `captureException` on share URL decode failures and localStorage errors
- `components/MapLibreMap.tsx` — Add `captureException` on map load failures

### Notes

- Run `npx @sentry/wizard@latest -i nextjs` first — it auto-generates the three config files and patches `next.config.js`. Do not create those files manually.
- The wizard also creates a test page (`/sentry-example-page`) to verify errors are being captured. Use it before wiring up custom instrumentation.
- All Sentry imports use `import * as Sentry from '@sentry/nextjs'` — the same package works on both client and server in Next.js.
- Do not log book titles, author names, or ISBN values in Sentry events — they add noise and are unnecessary for debugging.
- Alert rules (Task 5) are configured in the Sentry web dashboard, not in code.
